## 三、核心算法逻辑

### 3.1 五维人格属性（初始值 50，范围 0-100）

| 维度 | 英文键 | 含义 | 映射到波点 |
|------|--------|------|-----------|
| 占位度 | `presence` | 社交存在感、表现欲、能量场 | 波点大小 |
| 高密感 | `density` | 思考深度、情感细腻度、精神内耗 | 波点密度 |
| 秩序力 | `order` | 逻辑、强迫症倾向、对规则的依赖 | 波点排列 |
| 呼吸率 | `margin` | 边界感、松弛感、个人空间需求 | 波点间距 |
| 浮游感 | `fluctuation` | 跳跃思维、情绪波动、不可预测性 | 波点动态 |

### 3.2 题目数据结构

```js
{
  id: 1,
  title: "Q1_TITLE",           // ← 待填充
  options: [
    { label: "A", text: "Q1_OPT_A", weight: { presence: +10, density: 0, order: +5, margin: -3, fluctuation: -5 } },
    { label: "B", text: "Q1_OPT_B", weight: { presence: -5, density: +8, order: +3, margin: +5, fluctuation: 0 } },
    { label: "C", text: "Q1_OPT_C", weight: { presence: 0, density: -5, order: -8, margin: +10, fluctuation: +10 } },
  ]
}
```

每个选项的 `weight` 对象为该选项对五维的加减分（范围约 -15 ~ +15）。

### 3.3 积分累加器（Calculator 模块）

```
初始化: 五维均为 50
每次答题: scores[dim] += weight[dim] → clamp(0, 100)
最终: 得到五维最终得分
```

### 3.4 结果路由引擎（routeResult 函数）

判定优先级（从高到低）：

1. **极端高判定**：某一维度 > 80，结合辅助维度匹配特定类型
2. **极端低判定**：某一维度 < 20，匹配缺失型
3. **双高组合**：前两名维度特定组合（如 presence+fluctuation → 波普爆炸）
4. **均衡判定**：所有维度在 40-60 → 极简禅意
5. **兜底**：按最高维度映射到对应类型

### 3.5 8 种波点结果类型

| # | 类型名称 | 关键维度特征 | 波点视觉特征 |
|---|---------|-------------|------------|
| 1 | 巨型无序波点 | 高占位 + 低秩序 | 大点、混沌排列、高动态 |
| 2 | 经典等距波点 | 高秩序 + 中占位 | 中号、网格排列、低动态 |
| 3 | 微型高密波点 | 高密度 + 低占位 | 微小、极高密度、半网格 |
| 4 | 呼吸感留白波点 | 高呼吸率 + 低密度 | 中号、宽间距、低动态 |
| 5 | 动态异形波点 | 高浮游 + 低秩序 | 多变大小、随机、极高动态 |
| 6 | 立体浮雕波点 | 高占位 + 高秩序 | 大号、分层网格、中动态 |
| 7 | 极简禅意波点 | 均衡型（40-60） | 小号、稀疏网格、极低动态 |
| 8 | 波普爆炸波点 | 高占位+高浮游+高密 | 大号多变、极高密度、极高动态 |
